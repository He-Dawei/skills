# JD 分析规则详解

## 一、核心能力提取规则

从 JD 中提取最核心的 3 项能力要求。这 3 项能力是简历必须重点呼应的内容。

### 提取优先级
1. **"岗位要求""任职资格""必备技能"** 段落的前 3 条
2. 重复出现 2 次以上的技能/能力关键词
3. 措辞中使用"必须""需要""要求"等强约束词的条目

### 提取原则
- 每项能力必须附 **JD 原文依据**（引用原文句子）
- 按重要性排序，出现位置越靠前权重越高
- 优先提取可验证的硬技能（编程语言、框架、工具），而非泛化描述（"有责任心"）

### 输出格式

```
核心能力1：[能力名称] — [JD原文依据，引用1-2句]
核心能力2：[能力名称] — [JD原文依据，引用1-2句]
核心能力3：[能力名称] — [JD原文依据，引用1-2句]
```

### 示例

```
核心能力1：Java后端开发 — "熟悉Java/Spring Boot微服务架构，有高并发系统开发经验"
核心能力2：数据库设计与优化 — "熟悉MySQL，了解Redis等NoSQL数据库，具备SQL调优经验"
核心能力3：分布式系统 — "了解分布式系统原理，熟悉微服务治理、消息队列等中间件"
```

## 二、隐藏偏好推断规则

从 JD 的措辞、公司文化描述、团队介绍中推断 2 个 **未明说但隐含** 的招聘偏好。

### 推断来源

| 来源 | 典型措辞 | 隐含偏好 |
|------|----------|----------|
| 公司/团队文化 | "扁平化管理" | 自驱力、跨团队协作 |
| 公司/团队文化 | "快速迭代""敏捷开发" | 抗压能力、效率意识 |
| 公司/团队文化 | "从0到1""开拓新业务" | 开拓能力、创业精神 |
| 岗位职责 | "跨部门沟通" | 协调能力、表达能力 |
| 岗位职责 | "独立负责" | 独立工作能力、Owner 意识 |
| 加分项 | "有XX经验优先" | 实际很看重，可能是核心筛选标准 |
| 加分项 | "有开源项目/技术博客" | 技术热情、影响力 |
| 行业背景 | "电商/SaaS/金融" | 行业认知、业务理解力 |

### 推断原则
- 每项偏好必须附 **推断依据**（引用 JD 原文或段落）
- 避免过度解读：只推断有文本支撑的偏好
- 偏好应转化为简历中可以体现的具体行为或经历

### 输出格式

```
隐藏偏好1：[偏好描述] — [推断依据]
隐藏偏好2：[偏好描述] — [推断依据]
```

### 示例

```
隐藏偏好1：Owner意识与主动性 — "独立负责XX模块的设计与落地"暗示需要能独立扛事的人
隐藏偏好2：技术影响力 — "有开源项目或技术博客优先"实际是筛选标准之一
```

## 三、关键词提取规则

从 JD 中提取 10 个高频/高权重关键词，这些关键词必须在简历中有自然体现。

### 提取类别

| 类别 | 示例 |
|------|------|
| 技术栈 | Java, Python, Spring Boot, React, MySQL, Redis, Docker |
| 方法论/流程 | 敏捷开发, CI/CD, Code Review, 数据驱动, A/B测试 |
| 业务领域 | 电商, 支付, SaaS, 出行, 社交, 内容推荐 |
| 能力素质 | 跨团队协作, 项目管理, 技术方案设计, 问题排查 |

### 提取优先级
1. 技术栈名词（编程语言、框架、工具名）— 最高权重
2. 方法论/流程名词 — 高权重
3. 业务领域名词 — 中权重
4. 能力素质名词 — 低权重（除非出现 2 次以上）

### 去重与排序
- 按出现频率排序，频率相同则按重要性（技术栈 > 方法论 > 业务 > 素质）
- 去重后取前 10 个
- 同一概念的不同表述合并（如 "Java" 和 "java"）

### 输出格式

```
关键词：关键词1, 关键词2, 关键词3, ..., 关键词10
```

### 示例

```
关键词：Java, Spring Boot, MySQL, Redis, 微服务, 分布式, Kafka, Docker, CI/CD, 高并发
```

